# CityPulse - Dynamic Urban Heat & Bounty System

A production-ready FiveM resource with a three-dimensional "heat" system, NPC witness propagation, configurable AI police, **automatic NPC bounty generation**, and darknet bounty hunting through an LB Phone-style UI.

---

## Features

### Three-Dimensional Heat System
- **Violence Index** – gunshots, explosions, melee kills
- **Witness Index** – NPC witnesses, CCTV cameras, ALPR hits
- **Defiance Index** – ramming police, fleeing roadblocks, shooting officers
- Heat formula: `currentHeat = (violence × 0.4) + (witness × 0.3) + (defiance × 0.3)`
- 5 heat levels: Green → Yellow → Orange → Red → Black

### NPC Bounty System (NEW)
- **Automatic NPC crime generation** at configurable intervals (default: 5 minutes)
- 6 crime types: armed robbery, assault, murder, carjacking, drug dealing, police assault
- NPCs spawn at random locations with random names and models
- Hunters can accept and track NPC bounties just like player bounties
- NPC bounties expire after 10 minutes if unclaimed
- Configurable via `Config.EnableNPCCrimes`, `Config.NPCCrimeInterval`, `Config.MaxActiveNPCBounties`

### Configurable AI Police (NEW)
- **Master toggle**: `Config.EnableAIPolice = false` (disabled by default)
- When enabled: police vehicles, helicopters, SWAT, and roadblocks spawn based on heat level
- When disabled: heat system, CCTV, witnesses, and bounties still function normally
- Separate `Config.PoliceSettings` for fine-tuning response behavior

### NPC Witness System
- NPCs within range detect crimes and enter fear animations
- Witness state propagates to nearby NPCs (panic spreading)
- Per-NPC cooldown (30s) and area saturation cooldown prevent spam

### Darknet Bounty Hunting
- Player bounties auto-post when reaching Red/Black heat and escaping
- NPC bounties auto-generated by the crime system
- LB Phone-style cyberpunk UI with swipeable bounty cards
- Heatmap circle, heartbeat audio, photo confirmation, kill/subdue mechanics

### Error Handling & Stability (NEW)
- **Centralized error handler** (`shared/errorHandler.lua`) with pcall wrappers
- **React ErrorBoundary** prevents UI crashes from propagating
- **Safe NUI callbacks** – all callbacks wrapped to prevent NUI hanging
- Debug logging system with configurable verbosity

### Performance Optimization (NEW)
- `Config.Performance` section with tunable thread intervals
- `Config.Debug = false` by default (disable verbose logging in production)
- `React.memo` on expensive components (BountyCard)
- Entity garbage collection for stale police/NPC entities
- Rate limiting on all client-to-server events

## Installation

### Prerequisites
- FiveM server (recommended: latest artifacts)
- **oxmysql** or **ghmattimysql** (for persistent storage)
- **lb-phone** (for phone app integration)
- Node.js 18+ (for building the UI)

### Steps

1. **Copy the resource** into your server's `resources/` folder:
   ```
   resources/[citypulse]/citypulse/
   ```

2. **Import the database schema** (if using persistent storage):
   ```bash
   mysql -u root -p your_database < citypulse.sql
   ```
   > **Note**: If upgrading from a previous version, run these ALTER statements:
   > ```sql
   > ALTER TABLE citypulse_bounties ADD COLUMN is_npc TINYINT(1) DEFAULT 0;
   > ALTER TABLE citypulse_bounties ADD COLUMN npc_model VARCHAR(100) DEFAULT NULL;
   > ALTER TABLE citypulse_bounties ADD COLUMN failure_count INT DEFAULT 0;
   > ALTER TABLE citypulse_bounties ADD COLUMN transferred_to_police TINYINT(1) DEFAULT 0;
   > ALTER TABLE citypulse_bounties ADD INDEX idx_npc (is_npc);
   > ```

3. **Build the UI**:
   ```bash
   cd resources/[citypulse]/citypulse/ui
   npm install
   npm run build
   ```

4. **Configure** `config.lua`:
   - `Config.Framework` → `'standalone'`, `'esx'`, or `'qbcore'`
   - `Config.UseDatabase` → `true` if using MySQL
   - `Config.Language` → `'cn'` (Chinese) or `'en'` (English)
   - `Config.EnableNPCCrimes` → `true` to enable automatic NPC bounties
   - `Config.EnableAIPolice` → `false` (default) or `true` to enable police spawning

5. **Add to server.cfg**:
   ```cfg
   ensure citypulse
   ```

## Configuration Guide

### Key Config Sections

| Section | Description |
|---------|-------------|
| `Config.EnableNPCCrimes` | **NEW** – Toggle automatic NPC bounty generation |
| `Config.EnableAIPolice` | **NEW** – Toggle AI police spawning (off by default) |
| `Config.Performance` | **NEW** – Thread intervals and entity limits |
| `Config.Heat` | Weights, thresholds, decay rates, crime gain values |
| `Config.Districts` | 12 police districts with center/radius |
| `Config.Witness` | NPC detection/propagation/report parameters |
| `Config.Police` | Response per heat level, vehicle models, ALPR |
| `Config.CCTV` | Camera positions, scan interval, FOV |
| `Config.Bounty` | Rewards, timers, hunter limits, drop-off points |
| `Config.Disguise` | Mask system, face-known decay |
| `Config.UI` | Command name, item name, NUI focus |
| `Config.AntiExploit` | Rate limiting, kick thresholds |

### NPC Crime Configuration

```lua
Config.EnableNPCCrimes    = true    -- Enable/disable NPC bounty generation
Config.NPCCrimeInterval   = 300    -- Seconds between NPC crimes (5 minutes)
Config.MaxActiveNPCBounties = 10   -- Maximum concurrent NPC bounties
Config.NPCBountyExpiryTime = 600   -- NPC bounty expiry (10 minutes)
```

### AI Police Toggle

```lua
Config.EnableAIPolice = false  -- Set to true to enable police spawning
-- When false: heat system, CCTV, witnesses, and bounties still work
-- When true: police vehicles, helicopters, SWAT, and roadblocks spawn
```

### Performance Tuning

```lua
Config.Performance = {
    UpdateIntervals = {
        heatDecay    = 10000,  -- Increase to reduce server CPU
        npcWitness   = 2000,   -- Witness processing interval
        policeAI     = 500,    -- Only used if AI police enabled
    },
    MaxTrackedNPCs   = 50,     -- Reduce for lower-end servers
    MaxStoredBounties = 100,   -- Maximum bounties in memory
}
Config.Debug = false  -- ALWAYS disable in production
```

## Architecture Notes

### Performance
- All heavy calculations run server-side; client only detects and reports
- No per-frame loops except subdue key check (sleeps 500ms when idle)
- Witness processing: 2s intervals with 50-NPC cap per player
- Entity garbage collection every 30 seconds
- NPC bounty generation uses configurable intervals (default 5 min)

### Error Handling
- `CityPulseError.SafeExecute(fn, context)` – pcall wrapper with logging
- `CityPulseError.DebugPrint(module, msg)` – conditional debug output
- All NUI callbacks wrapped with `safeNUICallback()` to prevent hanging
- React `ErrorBoundary` catches render errors and shows fallback UI

### Anti-Exploit
- Server-side rate limiting (max 5 events/second per player)
- Kick threshold for repeated violations
- Server-validated bounty acceptance with daily limits

## Troubleshooting

| Issue | Solution |
|-------|----------|
| No bounties appearing | Check `Config.EnableNPCCrimes = true` and wait for interval |
| Police not spawning | Check `Config.EnableAIPolice = true` |
| UI not loading | Run `npm run build` in `ui/` folder, check `ui/dist/` exists |
| Database errors | Run `citypulse.sql` or ALTER statements for upgrades |
| High CPU usage | Increase `Config.Performance.UpdateIntervals` values, set `Config.Debug = false` |
| NUI hanging | This should be fixed by the safeNUICallback wrapper; check console for errors |
